# 怎么把这个 APP 拿给别人看

六种方式。想「随时随地发个链接给别人」看方式零点五；想「答辩现场最稳」看方式一。

---

## 方式零点五：在线链接（随时随地，发给谁都能开）

已发布，链接：

**https://claude.ai/code/artifact/4979c2a0-a11a-42e0-bed9-df79e80efa42**

- 手机、电脑浏览器都能直接打开，不用装任何东西。
- **默认是私有的。** 必须在该页面右上角的分享菜单里点一次分享，别人才打得开——不然对方会看到无权限。
- 整个应用（React + three.js + 全部样式）内联在一个页面里，零外部请求，加载后即使断网也能继续操作。

要更新内容：

```bash
npm run build:artifact
```

生成 `dist-single/artifact.html`，再让我用同一个文件路径重新发布，链接保持不变。

> 注意：这个页面是刻意锁死在浅色医疗风主题的（`scripts/build-artifact.mjs` 里的 themeLock），不会跟着查看者的深色模式反转。

## 方式零：安装成 Android APK（真机 APP，观感最好）

```bash
npm run build:apk
```

产物：`release/脑机康复闭环演示.apk`（约 3.3 MB）

- 包名 `com.bci.rehab.demo`，应用名「脑机康复闭环演示」，已用正式密钥签名（非 debug 包）。
- 最低支持 **Android 7.0（API 24）**，目标 API 36。
- 网页资源全部打进 APK，**安装后完全离线**，不联网。
- 装进真机后会自动隐藏页面里那套模拟状态栏和 home indicator（Android 会画自己的），手机壳铺满整屏。
- 因为没有上架应用商店，安装时手机会提示「未知来源」，需要手动允许。

技术实现是 Capacitor（Android WebView 壳）。三个已经踩过的坑，写在这里免得以后再撞：

1. **项目路径含中文**，Android Gradle Plugin 会直接拒绝构建。已在 `android/gradle.properties` 里加了 `android.overridePathCheck=true`（现代 aapt2 能处理 UTF-8 路径）。
2. **`maven.google.com` 在国内网络完全不通**（实测 20s 超时）。已把 `android/build.gradle` 的仓库指向阿里云镜像，Gradle 发行包指向腾讯镜像。这只是项目内的仓库配置，没有改任何系统代理。
3. **Gradle 缓存和 Android SDK 都放在纯 ASCII 路径**下（`D:\gradle-home`、`D:\Android\Sdk`），因为用户目录含中文。`npm run build:apk` 会自动带上 `GRADLE_USER_HOME`。

签名密钥在 `D:\Android\keystores\bci-release.jks`，密码写在 `android/keystore.properties`（已 gitignore）。**这个文件要自己备份**——换密钥重新签名的 APK 无法覆盖安装旧版本。

两个已知限制：

- APK 里**用不了 `?tab=` / `?step=` 深链**（Capacitor 从 `https://localhost/` 加载，没有地址栏）。演示要跳转就在应用内点。
- 应用图标目前还是 Capacitor 的默认图标，没做自定义。

## 方式一：单文件 HTML（推荐用于答辩、发给评委/老师）

把整个应用（含 React、three.js、全部样式）打进**一个 .html 文件**，无需网络、无需 Node、无需服务器，双击就能打开。

```bash
npm run build:single
```

产物：`dist-single/脑机协同康复机械臂闭环可视化系统.html`（约 747 kB）

- 直接微信 / QQ / 邮件发给别人，对方双击即可打开（Chrome / Edge 均可）。
- 拷进 U 盘带到答辩现场，断网也能跑。
- 深链参数在 `file://` 下同样有效，例如在地址栏末尾加 `?tab=ssvep`。

> 原理：单文件构建用的是 `vite.single.config.ts`（iife 格式），再由 `scripts/inline-single.mjs` 把 JS/CSS 内联进 HTML。
> 必须是 iife 而非 ES module——`file://` 下浏览器会按 CORS 规则拉取 module script，一定会被拦截。

## 方式二：手机上看（最像真 APP，适合现场演示）

应用本来就是按手机屏设计的（420×880 手机壳，窄屏自动全屏）。让手机和电脑连**同一个 WiFi**，然后：

```bash
npm run dev:lan
```

Vite 会打印出若干个 `Network:` 地址，用手机浏览器打开其中一个。

需要注意的两点：

1. **选对 IP。** 这台机器上有多个网卡（VMware / VirtualBox 会各占一个 `192.168.x.1`）。请挑选与手机 WiFi 处于同一网段的那个地址——通常不是 `192.168.x.1` 结尾为 `.1` 的那个。
2. **Windows 防火墙**第一次会弹窗询问是否允许 Node.js 访问网络，勾选「专用网络」并允许，否则手机连不上。

用 `npm run preview:lan` 可以同样方式预览生产构建（速度更快、更接近最终效果）。

## 方式三：发一个公网链接（GitHub Pages / Vercel / Netlify / Cloudflare Pages）

```bash
npm run build     # 产物在 dist/
```

`vite.config.ts` 里已经设成 `base: './'`（相对路径），所以 `dist/` 可以直接丢到任意静态托管的**任意子目录**，例如 GitHub Pages 的 `https://用户名.github.io/仓库名/`，不需要再改配置。已实测子目录下资源可正常加载。

本项目目前**还不是 git 仓库**。要走这条路需要先 `git init` 并推到 GitHub，这一步涉及你的账号，我没有替你执行。

## 答辩演示的深链清单

直接在地址栏末尾追加，可跳过点击流程直达任意画面：

| 链接参数 | 效果 |
| --- | --- |
| `?tab=home` | 首页（关键指标） |
| `?tab=demo` | 七步闭环训练页 |
| `?tab=demo&step=5` | 直接跳到「机器人动作响应」（3D 机械臂） |
| `?tab=ssvep` | SSVEP 被动接受范式页 |
| `?tab=ssvep&gaze=握拳` | SSVEP 页自动完成注视 → 解码，直接显示结果 |
| `?tab=report` | 训练报告页 |
| `?tab=system` | 技术架构与扩展能力页 |

`step` 取 1–7，`gaze` 取 抬手 / 伸肘 / 握拳 / 放松。

> **`?gaze=` 会直接开启 8–15 Hz 闪烁**（视为演示者已确认无光敏风险），因此这条链接适合自己答辩时用，不要直接甩给不清楚是否有光敏性癫痫风险的人。给别人发链接请用 `?tab=ssvep`，由对方自己点「开始闪烁刺激」。

> `gaze=握拳` 含中文，粘贴进聊天软件时可能被转成 `%E6%8F%A1%E6%8B%B3`，这是正常的 URL 编码，效果一致。

## 已经做过的兼容性处理

- **WebGL 不可用不会白屏。** 若对方电脑关闭了硬件加速或 GPU 被驱动黑名单拦截，3D 机械臂会降级为一块静态说明面板，训练页其余部分（轨迹图、控制误差、抓握力度）照常工作。整个应用还额外包了一层错误边界，任何子树报错只降级该子树。
- **光敏安全。** SSVEP 页的 8–15 Hz 闪烁**默认关闭**，需要访问者手动点「开始闪烁刺激」；同时遵循系统的 `prefers-reduced-motion` 设置。
- **完全离线。** 应用不请求任何外部资源、不联网、无字体/图片外链。
