在这个项目里面我打算完成一个APP的设计
这个APP的设计是用来参加这个大学竞赛，关于这个揭榜挂帅比赛

要求：
功能加上 普适性的扩展 +机器人多重形式的情绪安抚（多种形式表现在机器人的表情、语言 姿势 唱歌 跳舞 ）+奖励性增强功能（比如某种增强在多人中表现不错，那么记忆功能启动，在新来的个人中给与较大的权重，达到个体自适应）

把网页主流程改成比赛要求的闭环：脑电信号模拟 → 运动意图识别 → 光电感知校验 → AI康复决策 → 机器人动作响应 → 触觉反馈 → 训练报告生成。
具体修改重点：
首页
展示项目名称、系统定位和关键指标：意图识别准确率≥85%、响应延迟≤200ms、控制精度≤5mm、抓握力度误差≤10%。
保留现有光电模块，但改写用途
现有 RGB/NIR/ToF、Zero-DCE、YOLO、R(2+1)D-BERT 内容继续使用，但从“陪护看护”改为“康复动作识别与安全校验”，重点展示上肢姿态识别、弱光训练环境识别、动作轨迹偏差检测。
强化脑电交互模块
现有 EEG 波形不要只作为状态展示，要增加“抬手、伸肘、握拳、放松”四类运动意图按钮，点击后动态更新 EEG 波形、置信度、识别结果和误操作风险。
新增 AI康复决策模块
根据意图识别结果、光电姿态评分、疲劳指数，自动生成训练方案，例如：辅助抓握、标准力度、中速执行、压力+振动反馈、开启过力保护。
改造机器人动作响应模块
将现有机器人动画或骨架模块改成三个核心训练动作：肩部辅助抬升、肘部屈伸、手部抓握。每个动作要显示目标轨迹、实际轨迹、控制误差和执行状态。
新增触觉反馈闭环模块
展示压力反馈、振动反馈、位置感反馈三类反馈，并显示反馈强度、反馈延迟、安全阈值，体现“感觉运动闭环”。
新增训练报告生成模块
点击“生成报告”后展示本次训练结果：训练时长、完成次数、意图识别准确率、平均响应延迟、控制精度误差、抓握力度误差、疲劳指数变化和康复建议。
输出材料
演示流程固定为：开始训练 → 选择握拳意图 → EEG识别 → 光电校验 → AI决策 → 机器人抓握 → 触觉反馈 → 生成训练报告。
